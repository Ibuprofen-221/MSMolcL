# 代码变更报告 Skill

## 触发时机

当你完成一次代码编辑任务（修改、新增、修复）后，自动生成报告。如果你只是回答问题、查看代码、给出建议而没有实际修改代码，**不要**生成报告。

## 报告存放

将报告写入项目根目录 `/root/web/report/` 下，文件名为 `NNNN-任务简述.md`，其中：

- **NNNN**：4 位递增编号，从 0001 开始，查看 /report 目录下已有最大编号 +1
- **任务简述**：简短的任务摘要，不超过 10 个字，只写核心动作，不要修饰词

示例：`0005-修复登录限流漏洞.md`

## 报告格式（必须严格遵循）

```markdown
# [标题：编号-任务简述]

## 目的

[一句话说明为什么要做这次修改，解决什么问题]

## 变更

| 文件 | 类型 | 说明 |
|------|------|------|
| path/to/file.py | 修改/新增/删除 | 一句话描述具体改了什么 |

## 完成情况

[1-2 句话概括完成度]
```

## 约束

- **总长度不超过 50 行**。报告是用来快速回溯的，不是工作日志。
- **不写详细执行步骤**。Git diff 能看到具体改动。
- **不写推测和建议**。只记录实际发生的变更。
- **不重复粘贴代码**。描述即可。
- 如果 `/root/web/report/` 目录不存在，先创建它。
