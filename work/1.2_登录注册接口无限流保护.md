# 1.2 登录/注册接口无限流保护 — 解决方案

## 问题定位

`backend/api/auth.py` 中 `/api/register` 和 `/api/login` 两个接口均未添加 `@limiter.limit` 装饰器，可被暴力破解枚举用户名和密码。

```python
# backend/api/auth.py:16-71（当前代码）

@auth_router.post("/register", status_code=status.HTTP_200_OK)
async def register_api(payload: RegisterRequest, db: Session = Depends(get_db)):
    # 无限流保护

@auth_router.post("/login", status_code=status.HTTP_200_OK)
async def login_api(payload: LoginRequest, db: Session = Depends(get_db)):
    # 无限流保护
```

## 影响

- 攻击者可暴力枚举用户名和密码
- 无登录失败递增延迟，自动化爆破成本极低

## 解决方案

### 1. 新增配置项

在 `backend/core/config.py` 添加 auth 专用限流规则：

```python
# backend/core/config.py（新增）

# Auth rate limiting
rate_limit_auth_rule = "5/minute"              # 登录/注册: 每分钟 5 次
rate_limit_auth_block_seconds = 300             # 连续失败 10 次后封禁 5 分钟
auth_max_failed_attempts = 10                   # 触发封禁的失败次数
```

### 2. 对 auth 接口添加限流

修改 `backend/api/auth.py`：

```python
# backend/api/auth.py
from core.config import rate_limit_auth_rule
from core.rate_limit import limiter

@auth_router.post("/register", status_code=status.HTTP_200_OK)
@limiter.limit(rate_limit_auth_rule)          # 新增
async def register_api(payload: RegisterRequest, db: Session = Depends(get_db)):
    ...

@auth_router.post("/login", status_code=status.HTTP_200_OK)
@limiter.limit(rate_limit_auth_rule)          # 新增
async def login_api(payload: LoginRequest, db: Session = Depends(get_db)):
    ...
```

### 3. 添加登录失败计数与递增延迟

在 `backend/core/` 下新增 `auth_guard.py`：

```python
# backend/core/auth_guard.py
import time
from collections import defaultdict
from threading import Lock

from core.config import auth_max_failed_attempts, rate_limit_auth_block_seconds


class LoginGuard:
    """基于 IP + 用户名的登录失败计数，超阈值返回等待秒数。"""

    def __init__(self):
        self._lock = Lock()
        # key: (ip, username) → list[timestamp]
        self._failures: dict[tuple[str, str], list[float]] = defaultdict(list)

    def record_failure(self, ip: str, username: str) -> int | None:
        """记录一次失败。返回 None 表示未触发封禁；返回 int 表示需等待秒数。"""
        now = time.time()
        key = (ip, username)

        with self._lock:
            # 清理过期的失败记录（超过封禁窗口）
            cutoff = now - rate_limit_auth_block_seconds
            self._failures[key] = [t for t in self._failures[key] if t > cutoff]
            self._failures[key].append(now)

            if len(self._failures[key]) >= auth_max_failed_attempts:
                return rate_limit_auth_block_seconds
        return None

    def clear_on_success(self, ip: str, username: str):
        key = (ip, username)
        with self._lock:
            self._failures.pop(key, None)


login_guard = LoginGuard()
```

修改 `backend/api/auth.py` 的 login 函数：

```python
# backend/api/auth.py（修改 login_api）
from core.auth_guard import login_guard

@auth_router.post("/login", status_code=status.HTTP_200_OK)
@limiter.limit(rate_limit_auth_rule)
async def login_api(payload: LoginRequest, request: Request, db: Session = Depends(get_db)):
    username = (payload.username or "").strip()
    password = payload.password or ""

    # 检查是否已触达封禁阈值
    client_ip = request.client.host if request.client else "unknown"
    wait_seconds = login_guard.record_failure(client_ip, username)
    if wait_seconds:
        raise HTTPException(
            status_code=429,
            detail=f"登录失败次数过多，请 {wait_seconds // 60} 分钟后重试",
        )

    user = db.query(User).filter(User.username == username).first()
    if user is None or not verify_password(password, user.hashed_password):
        raise HTTPException(status_code=401, detail="用户名或密码错误")

    # 登录成功：清除失败记录
    login_guard.clear_on_success(client_ip, username)

    access_token = create_access_token(subject=user.username)
    return success_response(
        message="登录成功",
        data={"access_token": access_token, "token_type": "bearer", "username": user.username},
    )
```
