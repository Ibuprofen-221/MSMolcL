# 1.5 `torch.load(weights_only=False)` 任意代码执行 — 解决方案

## 问题定位

三处 `torch.load` 使用了 `weights_only=False`，允许任意 pickle 反序列化：

```python
# backend/services/retrieve.py:658
checkpoint = torch.load(model_weight_path, map_location="cpu", weights_only=False)

# backend/services/retrieve_advanced.py:106
checkpoint = torch.load(model_weight_path, map_location="cpu", weights_only=False)

# backend/services/model/predict.py:223
checkpoint = torch.load(model_weight_path, map_location="cpu", weights_only=False)
```

PyTorch 的 pickle 反序列化可执行任意 Python 代码。若模型文件被替换为恶意构造的 `.pth`，加载时即触发 RCE。

## 影响

- **严重程度: CRITICAL**
- 攻击者如能写入模型文件路径（文件系统入侵、依赖投毒、供应链攻击），即可获得服务器 Shell。

## 解决方案

### 方案选择

PyTorch 1.13+ 开始支持 `weights_only=True`，但该模式**仅加载张量数据**，不加载 Python 对象。需验证当前代码的 checkpoint 结构。

#### 分析 checkpoint 内容

当前 `retrieve.py` 和 `retrieve_advanced.py` 的 checkpoint 为自定义训练保存的完整模型（含 `state_dict` 和可能的额外元数据）。需要检查保存时使用的格式：

```python
# 检查现有保存逻辑使用的格式（通常在 training 脚本中）
# 如果是 torch.save(model.state_dict(), path) —— 纯 state_dict，可安全使用 weights_only=True
# 如果是 torch.save({'model': model.state_dict(), 'optimizer': ..., 'epoch': n}, path) —— 包含非张量对象，需改造
```

### 实施步骤

#### Step 1: 统一使用安全加载函数

新建 `backend/services/model_loader.py`：

```python
# backend/services/model_loader.py
import hashlib
import json
from pathlib import Path

import torch


def compute_file_hash(path: str | Path, algorithm: str = "sha256") -> str:
    """计算文件哈希。"""
    h = hashlib.new(algorithm)
    with open(path, "rb") as f:
        while chunk := f.read(8192):
            h.update(chunk)
    return h.hexdigest()


def verify_model_checksum(model_path: str | Path, manifest_path: str | Path) -> bool:
    """对比模型文件与清单中的预期哈希。"""
    manifest = json.loads(Path(manifest_path).read_text(encoding="utf-8"))
    expected = manifest.get(Path(model_path).name)
    if not expected:
        return False
    actual = compute_file_hash(model_path)
    return actual == expected


def safe_load_model(model_path: str | Path, map_location: str = "cpu") -> dict:
    """
    安全加载 PyTorch 模型。

    优先使用 weights_only=True。若模型结构复杂（含非张量对象），
    则先校验文件完整性，再用 weights_only=False 加载。
    """
    model_path = Path(model_path)

    # 优先尝试 weights_only=True（PyTorch >= 1.13）
    try:
        checkpoint = torch.load(str(model_path), map_location=map_location, weights_only=True)
        return checkpoint
    except Exception:
        pass

    # 回退：有校验文件时才允许 weights_only=False
    manifest_path = model_path.parent / "checkpoint_manifest.json"
    if not manifest_path.exists():
        raise RuntimeError(
            f"模型文件 {model_path.name} 不存在校验清单，拒绝以 weights_only=False 加载"
        )

    if not verify_model_checksum(model_path, manifest_path):
        raise RuntimeError(
            f"模型文件 {model_path.name} 校验失败，文件可能被篡改"
        )

    checkpoint = torch.load(str(model_path), map_location=map_location, weights_only=False)
    return checkpoint
```

#### Step 2: 生成校验清单

为现有模型文件生成 SHA-256 哈希清单：

```bash
# 在模型目录执行一次（管理员操作）
python -c "
import hashlib, json
from pathlib import Path

dir = Path('autodl-tmp')
manifest = {}
for f in dir.glob('*.pth'):
    h = hashlib.sha256()
    h.update(f.read_bytes())
    manifest[f.name] = h.hexdigest()

(dir / 'checkpoint_manifest.json').write_text(json.dumps(manifest, indent=2))
print('Done:', len(manifest), 'files')
"
```

#### Step 4: 模型训练脚本同步改造

确保训练保存脚本仅保存 `state_dict`：

```python
# 推荐的保存方式（训练脚本中）
torch.save(model.state_dict(), save_path)
# 而非 torch.save({'model': model, 'epoch': ...}, save_path)
```

若必须保存额外元数据，将 `state_dict` 与元数据分开存储：

```python
torch.save(model.state_dict(), weights_path)
Path(meta_path).write_text(json.dumps({'epoch': epoch, 'loss': loss}))
```
