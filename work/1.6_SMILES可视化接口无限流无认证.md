# 1.6 SMILES 可视化接口无限流无认证 — 解决方案

## 问题定位

`backend/api/smiles_visualization.py` 中的 SMILES 渲染接口既无认证检查，也无 `@limiter.limit` 防护：

```python
# backend/api/smiles_visualization.py（当前代码）

@smiles_visualization_router.get("/smiles_visualization", status_code=status.HTTP_200_OK)
async def generate_molecule_image(
    smiles: str = Query(..., description="SMILES字符串"),
):
    """根据 SMILES 字符串生成2D分子结构图，返回图片URL。"""
    # 无限流、无认证 — 任何人均可调用
    # 调用 RDKit 渲染，消耗 CPU
```

## 影响

- 匿名攻击者可通过大量并发请求耗尽 CPU 资源（DoS 向量）
- RDKit 渲染本身有一定计算开销，恶意 SMILES 可能触发慢速渲染路径

## 解决方案

### 1. 添加认证依赖

给接口加上 `get_current_user` 依赖，与项目中其他接口保持一致：

```python
# backend/api/smiles_visualization.py（修改后）
from fastapi import APIRouter, Depends, Query, status
from core.auth import get_current_user
from core.rate_limit import limiter
from core.config import rate_limit_default_rule
from core.response import success_response
from models.user import User
from services.visualization import MoleculeVisualizer

smiles_visualization_router = APIRouter(prefix="/api", tags=["smiles_visualization"])

@smiles_visualization_router.get("/smiles_visualization", status_code=status.HTTP_200_OK)
@limiter.limit(rate_limit_default_rule)                     # 新增：限流
async def generate_molecule_image(
    smiles: str = Query(..., description="SMILES字符串"),
    current_user: User = Depends(get_current_user),          # 新增：认证
):
    ...
```

### 2. 添加 SMILES 输入校验与长度限制

在渲染前增加输入校验，防止恶意输入：

```python
# backend/api/smiles_visualization.py（新增校验函数）
import re

MAX_SMILES_LENGTH = 512  # 合理上限

_SMILES_PATTERN = re.compile(r"^[A-Za-z0-9@\[\]\(\)\\/#=+\-.,;:%\[\]]+$")

def _validate_smiles(smiles: str) -> None:
    safe = smiles.strip()
    if len(safe) > MAX_SMILES_LENGTH:
        raise HTTPException(status_code=400, detail=f"SMILES 字符串长度不能超过 {MAX_SMILES_LENGTH}")
    if not _SMILES_PATTERN.match(safe):
        raise HTTPException(status_code=400, detail="SMILES 字符串包含非法字符")
```

### 3. 添加 RDKit 渲染超时保护

为防止恶意 SMILES 触发超长渲染，使用 `asyncio.to_thread()` + 超时：

```python
import asyncio

@smiles_visualization_router.get("/smiles_visualization", status_code=status.HTTP_200_OK)
@limiter.limit(rate_limit_default_rule)
async def generate_molecule_image(
    smiles: str = Query(..., description="SMILES字符串"),
    current_user: User = Depends(get_current_user),
):
    _validate_smiles(smiles)

    # ...构建 target_path...

    try:
        # RDKit 渲染在线程池中执行，设置 5 秒超时
        await asyncio.wait_for(
            asyncio.to_thread(
                MoleculeVisualizer.save_molecule_2d,
                smiles=smiles.strip(),
                output_path=str(target_path),
            ),
            timeout=5.0,
        )
    except asyncio.TimeoutError:
        raise HTTPException(status_code=500, detail="分子结构渲染超时")
    except Exception:
        raise HTTPException(status_code=500, detail="分子结构渲染失败")

    if not target_path.exists():
        raise HTTPException(status_code=500, detail="分子结构图片生成失败")

    return success_response(
        data={"image_url": f"/smiles_image/{target_path.name}"},
        message="分子结构图生成成功",
    )
```

### 4. 添加渲染结果缓存

相同 SMILES 的渲染结果可复用，避免重复计算：

```python
# 在 services/visualization.py 或新增缓存层
from pathlib import Path
import hashlib

def _smiles_cache_key(smiles: str) -> str:
    return hashlib.sha256(smiles.strip().encode()).hexdigest()[:16]

# 在 generate_molecule_image 中使用 cache-key 命名文件：
cache_key = _smiles_cache_key(smiles)
target_path = smiles_image_dir / f"{cache_key}.png"

if target_path.exists():
    # 直接返回缓存
    return success_response(
        data={"image_url": f"/smiles_image/{target_path.name}"},
        message="分子结构图生成成功（缓存）",
    )
```

### 5. 配置独立限流规则（推荐）

为可视化接口使用更严格的限流，与通用接口分离：

```python
# backend/core/config.py（新增）
rate_limit_smiles_rule = "20/minute"  # SMILES 渲染比普通查询更耗 CPU
```
