# 1.8 Task ID 可猜测导致越权访问 — 解决方案

## 问题定位

当前 `_ensure_task_belongs_to_user` 仅检查 task 目录是否存在于当前用户的 `data_path` 下：

```python
# backend/api/retrieve.py:56-61
def _ensure_task_belongs_to_user(user_data_path: str, task_id: str) -> bool:
    try:
        task_dir = get_user_task_dir(user_data_path=user_data_path, task_id=task_id, create=False)
    except ValueError:
        return False
    return task_dir.exists() and task_dir.is_dir()
```

但这个检查依赖于 `task_id` 目录确实存在于该用户目录下。问题场景：
- Task ID 是 `uuid4().hex`（32 位 hex 字符串），虽难穷举，但一经泄露即永久可被访问
- AdvancedSearch 页面允许输入任意 `task_id`，只要目录存在即通过校验
- 没有任何机制验证当前用户是否是该 task 的创建者

```python
# backend/api/statas.py:39-46 — 同样仅依赖路径存在性
# backend/api/retrieve.py:397,411 — 同样仅依赖路径存在性
# backend/api/spectrum_detail.py:41 — 同样仅依赖路径存在性
```

## 影响

若用户通过 URL 分享、日志泄漏、截屏等方式暴露了 task_id，任何已认证用户均可访问该任务数据。

## 解决方案

### 核心思路

为每个 task 目录创建时写入一个 `task_owner.json` 文件，记录创建者用户名。每次访问时校验当前用户是否为 owner。

### Step 1: 新增 task 所有权标记

修改 `backend/util/file_utils.py`，在创建 task 目录时写入所有权文件：

```python
# backend/util/file_utils.py（修改 get_user_task_dir）

import json
from datetime import datetime, timezone

TASK_OWNER_FILENAME = "task_owner.json"

def _write_task_owner(task_dir: Path, username: str) -> None:
    """创建 task 所有权标记文件。"""
    owner_file = task_dir / TASK_OWNER_FILENAME
    owner_file.write_text(
        json.dumps({
            "owner": username,
            "created_at": datetime.now(timezone.utc).isoformat(),
        }, ensure_ascii=False, indent=2),
        encoding="utf-8",
    )

def get_user_task_dir(
    user_data_path: str, task_id: str,
    storage: str = "normal", create: bool = False,
    owner: str | None = None,  # 新增参数
) -> Path:
    _ = storage
    user_dir = resolve_user_data_dir(user_data_path)
    safe_task_id = _safe_task_id(task_id)

    task_dir = (user_dir / f"task_{safe_task_id}").resolve()
    try:
        task_dir.relative_to(user_dir)
    except ValueError:
        raise ValueError("非法task_id") from None

    if create:
        task_dir.mkdir(parents=True, exist_ok=True)
        if owner and not (task_dir / TASK_OWNER_FILENAME).exists():
            _write_task_owner(task_dir, owner)

    return task_dir
```

### Step 2: 新增统一的所有权校验函数

```python
# backend/util/file_utils.py（新增）

def verify_task_ownership(task_dir: Path, username: str) -> bool:
    """校验 task 目录的所有权。"""
    owner_file = task_dir / TASK_OWNER_FILENAME
    if not owner_file.exists():
        return False  # 无所有权文件，拒绝访问

    try:
        data = json.loads(owner_file.read_text(encoding="utf-8"))
    except Exception:
        return False

    return data.get("owner") == username
```

### Step 3: 增强 `_ensure_task_belongs_to_user`

修改 `backend/api/retrieve.py` 中的校验函数：

```python
# backend/api/retrieve.py（修改后）

from util.file_utils import verify_task_ownership

def _ensure_task_belongs_to_user(user_data_path: str, task_id: str, username: str) -> bool:
    """检查 task 目录存在且当前用户为创建者。"""
    try:
        task_dir = get_user_task_dir(
            user_data_path=user_data_path,
            task_id=task_id,
            create=False,
        )
    except ValueError:
        return False

    if not task_dir.exists() or not task_dir.is_dir():
        return False

    # 新增：所有权校验
    return verify_task_ownership(task_dir, username)
```

### Step 4: 更新所有调用点

```python
# backend/api/retrieve.py（各端点中更新调用）

# 之前：
if not _ensure_task_belongs_to_user(current_user.data_path, task_id):
    return JSONResponse(status_code=404, content=error_response(message="任务不存在或无权限", code=404))

# 之后：
if not _ensure_task_belongs_to_user(current_user.data_path, task_id, current_user.username):
    return JSONResponse(status_code=404, content=error_response(message="任务不存在或无权限", code=404))
```

同样的修改应用到：
- `backend/api/retrieve.py:397, 411, 504, 518`
- `backend/api/statas.py:39-68` 的 `get_statas` 函数
- `backend/api/spectrum_detail.py:30-52` 的 `get_spectrum_plot` 函数
- `backend/api/download_file.py` 的下载端点

### Step 5: 创建 task 时写入所有权

在创建 task 目录的入口传入 `owner` 参数：

```python
# backend/api/file_upload.py（创建 task_dir 处）
task_dir = get_user_task_dir(
    current_user.data_path, task_id, create=True,
    owner=current_user.username,  # 新增
)

# backend/services/sirius_batch_service.py（创建 task_dir 处）
task_dir = get_user_task_dir(
    user_data_path=user_data_path, task_id=task_id, create=True,
    owner=...  # 需从调用链传入
)
```

对于 sirius_batch_service，需要修改 `submit_mgf_only_batch` 接口签名增加 `owner` 参数，或在调用层通过 `current_user.username` 传入。

### 向后兼容处理

对于已有的无 `task_owner.json` 的 task 目录，首次访问时自动补写所有权文件：

```python
def _ensure_owner_file(task_dir: Path, username: str) -> bool:
    """兼容性处理：旧 task 目录没有所有权文件时，以当前用户写入。"""
    owner_file = task_dir / TASK_OWNER_FILENAME
    if not owner_file.exists():
        # 仅当 task 目录在用户 data_path 下时，才补写所有权
        _write_task_owner(task_dir, username)
        return True

    data = json.loads(owner_file.read_text(encoding="utf-8"))
    return data.get("owner") == username
```
