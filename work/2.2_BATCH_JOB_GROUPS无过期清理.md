# 2.2 BATCH_JOB_GROUPS 无过期清理 — 解决方案

## 问题定位

`backend/api/retrieve.py` 中 `BATCH_JOB_GROUPS` 是模块级字典，存储批次检索任务组信息，写入后永不清除：

```python
# backend/api/retrieve.py:28
BATCH_JOB_GROUPS: dict[str, dict] = {}

# backend/api/retrieve.py:186 — 写入
BATCH_JOB_GROUPS[group_id] = {...}

# backend/api/retrieve.py:394,501 — 读取
group = BATCH_JOB_GROUPS.get(job_id)
```

每次多文件检索任务提交时都会写入一条记录，且无任何淘汰机制。

## 影响

- 长期运行后内存泄漏
- 过期任务组数据占用内存且不可达

## 解决方案

### 方案选择

使用带 TTL 的字典替换普通 `dict`。有两种实现方式：

**方案 A**：使用 `cachetools.TTLCache`（推荐）

选择方案 A，因为更简洁且已有成熟实现。

### Step 1: 安装依赖或自行实现

`cachetools` 是纯 Python 包，无需额外系统依赖。如果不想引入新依赖，可自行实现精简版 TTL dict：

```python
# backend/api/retrieve.py（新增，如果不用 cachetools）
import time
from threading import Lock


class _TTLDict:
    """带 TTL 的字典，定期或访问时淘汰过期条目。"""

    def __init__(self, ttl_seconds: float = 7200):
        self._ttl = ttl_seconds
        self._data: dict[str, tuple[float, dict]] = {}
        self._lock = Lock()

    def get(self, key: str) -> dict | None:
        with self._lock:
            entry = self._data.get(key)
            if entry is None:
                return None
            created_at, value = entry
            if time.monotonic() - created_at > self._ttl:
                del self._data[key]
                return None
            return value

    def __setitem__(self, key: str, value: dict):
        with self._lock:
            self._data[key] = (time.monotonic(), value)

    def __delitem__(self, key: str):
        with self._lock:
            self._data.pop(key, None)

    def cleanup(self):
        """主动清理过期条目（可由定时任务调用）。"""
        now = time.monotonic()
        with self._lock:
            expired = [
                k for k, (ts, _) in self._data.items()
                if now - ts > self._ttl
            ]
            for k in expired:
                del self._data[k]
```

### Step 2: 替换 `BATCH_JOB_GROUPS`

```python
# backend/api/retrieve.py（修改后）

# 修改前
# BATCH_JOB_GROUPS: dict[str, dict] = {}

# 修改后
BATCH_GROUP_TTL_SECONDS = 7200  # 批次任务组保留 2 小时
BATCH_JOB_GROUPS = _TTLDict(ttl_seconds=BATCH_GROUP_TTL_SECONDS)
```

### Step 3: 在合并完成后主动清理

批次任务在合并完成后 (`_merge_batch_outputs`) 即可从缓存中移除，因为前端不再需要轮询：

```python
# backend/api/retrieve.py（修改 _merge_batch_outputs）

def _merge_batch_outputs(group: dict) -> None:
    if group.get("merged"):
        return

    # ... 合并逻辑不变 ...

    group["merged"] = True

    # 合并完成后延迟清理（给前端最后一次轮询机会）
    # 设置一个清理标记，下次 _cleanup 时删除
    group["_delete_after"] = time.monotonic() + 60  # 60 秒后删除
```

### Step 4: 添加定时清理任务

在 FastAPI lifespan 中启动后台清理协程：

```python
# backend/main.py（在 lifespan 中添加）

@asynccontextmanager
async def lifespan(_: FastAPI):
    init_db()
    await retrieve_runtime.startup()
    await retrieve_advanced_runtime.startup()

    # 新增：启动批次任务组清理协程
    cleanup_task = asyncio.create_task(_periodic_cleanup_batch_groups())

    try:
        yield
    finally:
        cleanup_task.cancel()
        await retrieve_advanced_runtime.shutdown()
        await retrieve_runtime.shutdown()


async def _periodic_cleanup_batch_groups(interval: int = 300):
    """每 5 分钟清理过期的批次任务组。"""
    from api.retrieve import BATCH_JOB_GROUPS

    while True:
        try:
            await asyncio.sleep(interval)
            BATCH_JOB_GROUPS.cleanup()
        except asyncio.CancelledError:
            break
        except Exception:
            pass
```

### Step 5: 在配置中集中 TTL 值

```python
# backend/core/config.py（新增）
batch_group_ttl_seconds = 7200  # 批次任务组 TTL：2 小时
batch_group_cleanup_interval = 300  # 清理间隔：5 分钟
```
