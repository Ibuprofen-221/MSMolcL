# 2.3 内存缓存无上限无过期 — 解决方案

## 问题定位

`backend/core/memory_store.py` 中的 `custom_lib_cache` 永不清除，且 `processed_cache` 在并发场景下可能被覆盖：

```python
# backend/core/memory_store.py:1-9

# 模块级可变字典 — 跨请求共享
processed_cache = {
    "valid_pairs_spectra_content": "",
    "valid_pairs_fragtrees_content": {},
    "statas": {},
}

custom_lib_cache: dict[str, list[str]] = {}
```

- `processed_cache` 每次调用 `update_processed_cache()` 时覆盖，但无任何并发保护
- `custom_lib_cache` 每次 `set_custom_lib_cache()` 追加一条，两处均无删除逻辑

```python
# backend/core/memory_store.py:19-26
def set_custom_lib_cache(file_name: str, smiles_list: list[str]) -> None:
    custom_lib_cache[file_name] = smiles_list  # 永不过期

def get_custom_lib_cache(file_name: str) -> list[str] | None:
    return custom_lib_cache.get(file_name)
```

调用链：
```python
# backend/api/candidates_choose.py:165
custom_lib_file_name = custom_lib_file.filename or "custom_lib.txt"
set_custom_lib_cache(custom_lib_file_name, smiles_list)  # 全量 SMILES 列表存入内存
```

## 影响

- 每次上传自定义库文件，完整 SMILES 列表永久驻留内存
- `processed_cache` 无并发保护，多请求下数据可能错乱
- 虽当前重名覆盖是设计意图，但缺乏 TTL 意味着无用的旧缓存永不释放

## 解决方案

### Step 1: 改造 `custom_lib_cache` 为任务级生命周期

自定义库缓存应绑定到 task 而非全局，随 task 生命周期自动释放：

```python
# backend/core/memory_store.py（重写）

import time
from threading import Lock

CUSTOM_LIB_TTL_SECONDS = 3600  # 1 小时

_custom_lib_lock = Lock()
_custom_lib_store: dict[str, tuple[float, list[str]]] = {}
# key: task_id:filename → (expire_at_timestamp, smiles_list)


def set_custom_lib_cache(task_id: str, smiles_list: list[str]) -> None:
    """
    将自定义库 SMILES 列表写入任务级缓存。
    每个 task 仅保留一份。
    """
    with _custom_lib_lock:
        # 清理过期条目（访问时顺便清理）
        _cleanup_expired_custom_libs_locked()

        key = f"task:{task_id}"
        expire_at = time.monotonic() + CUSTOM_LIB_TTL_SECONDS
        _custom_lib_store[key] = (expire_at, smiles_list)


def get_custom_lib_cache(task_id: str) -> list[str] | None:
    """读取任务级自定义库缓存。"""
    with _custom_lib_lock:
        entry = _custom_lib_store.get(f"task:{task_id}")
        if entry is None:
            return None

        expire_at, smiles = entry
        if time.monotonic() > expire_at:
            del _custom_lib_store[f"task:{task_id}"]
            return None

        return smiles


def remove_custom_lib_cache(task_id: str) -> None:
    """主动释放任务的自定义库缓存。"""
    with _custom_lib_lock:
        _custom_lib_store.pop(f"task:{task_id}", None)


def _cleanup_expired_custom_libs_locked() -> None:
    """清理所有过期的自定义库缓存（需在锁内调用）。"""
    now = time.monotonic()
    expired = [k for k, (expire_at, _) in _custom_lib_store.items() if now > expire_at]
    for k in expired:
        del _custom_lib_store[k]
```

### Step 2: 更新调用点

```python
# backend/api/candidates_choose.py（修改后）

# 修改前
custom_lib_file_name = custom_lib_file.filename or "custom_lib.txt"
set_custom_lib_cache(custom_lib_file_name, smiles_list)

# 修改后
set_custom_lib_cache(task_id, smiles_list)
custom_lib_file_name = custom_lib_file.filename or "custom_lib.txt"
```

对应的检索服务也需要从 `get_custom_lib_cache(task_id)` 读取，而非按文件名：

```python
# backend/services/retrieve_service.py（修改后，示例）

# 修改前
# custom_smiles = get_custom_lib_cache(job_payload.get("custom_lib_file_name"))

# 修改后
custom_smiles = get_custom_lib_cache(job_payload.get("task_id"))
```

### Step 3: 改造 `processed_cache` 加锁保护

```python
# backend/core/memory_store.py（追加修改）

_processed_lock = Lock()

def update_processed_cache(spectra_content: str, fragtrees_content: dict, statas: dict) -> None:
    """更新预处理结果内存缓存（线程安全）。"""
    with _processed_lock:
        processed_cache["valid_pairs_spectra_content"] = spectra_content
        processed_cache["valid_pairs_fragtrees_content"] = fragtrees_content
        processed_cache["statas"] = statas


def get_processed_cache() -> dict:
    """读取预处理缓存快照（线程安全）。"""
    with _processed_lock:
        return dict(processed_cache)
```

### Step 4: 配置外部化

```python
# backend/core/config.py（新增）
custom_lib_cache_ttl_seconds = 3600     # 自定义库缓存 TTL
```

## 变更文件清单

| 文件 | 操作 |
|------|------|
| `backend/core/config.py` | 新增 `custom_lib_cache_ttl_seconds` |
| `backend/core/memory_store.py` | 重写 `custom_lib_cache` 为任务级 TTL 缓存 + 锁保护；`processed_cache` 加锁 |
| `backend/api/candidates_choose.py` | 调用 `set_custom_lib_cache` 改为传 `task_id` |
| `backend/services/retrieve_service.py` | 调用 `get_custom_lib_cache` 改为传 `task_id` |
