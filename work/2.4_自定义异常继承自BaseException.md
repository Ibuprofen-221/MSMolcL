# 2.4 自定义异常继承自 BaseException — 解决方案

## 问题定位

`backend/core/exceptions.py` 中四个自定义异常全部继承自 `BaseException` 而非 `Exception`：

```python
# backend/core/exceptions.py（当前代码）

class FileMissingError(BaseException):   # line 1
    pass

class FileSizeError(BaseException):      # line 5
    pass

class FileFormatError(BaseException):    # line 9
    pass

class SiriusWorkerError(BaseException):  # line 13
    pass
```

Python 异常层次：

```
BaseException
├── SystemExit
├── KeyboardInterrupt
├── GeneratorExit
└── Exception
    ├── ValueError
    ├── RuntimeError
    └── ...所有正常业务异常...
```

`BaseException` 是为系统级退出信号预留的基类。自定义业务异常继承它将导致：

```python
# 任何 except Exception: 块都不会捕获这些异常
try:
    do_something()
except Exception:
    # FileMissingError 不会进入这里！
    pass
```

## 影响

当前代码在 `file_upload.py:342` 使用了显式捕获，因此暂时安全：

```python
except (FileMissingError, FileFormatError, FileSizeError) as exc:
    raise HTTPException(status_code=400, detail=str(exc)) from None
except Exception:
    raise HTTPException(status_code=500, detail="文件处理失败") from None
```

但若未来有开发者添加 `except Exception:` 通用处理逻辑，这些异常将被漏过，直接传播到 FastAPI 层返回 500。

## 解决方案

这是一个最小改动、最大安全收益的修复。

### 修改文件

```python
# backend/core/exceptions.py（修改后）

class FileMissingError(Exception):   # BaseException → Exception
    """必需文件缺失。"""
    pass


class FileSizeError(Exception):       # BaseException → Exception
    """文件大小超限。"""
    pass


class FileFormatError(Exception):     # BaseException → Exception
    """文件格式不合法。"""
    pass


class SiriusWorkerError(Exception):   # BaseException → Exception
    """Sirius Worker 异常。"""
    pass
```

### 无需修改调用点

由于所有现有 `raise` 和 `except` 均以类名显式引用，修改基类不影响调用方：

```python
# backend/api/file_upload.py:342 — 无需修改，继续正常工作
except (FileMissingError, FileFormatError, FileSizeError) as exc:
    ...

# backend/api/file_upload.py:344
except SiriusBatchError as exc:
    ...
```

### 验证清单

```bash
# 确认所有 raise 子句均使用明确的类名
grep -rn "raise FileMissingError\|raise FileSizeError\|raise FileFormatError\|raise SiriusWorkerError" backend/
# 预期结果：仅 backend/core/exceptions.py（定义处）和 call sites

# 确认没有 except BaseException 依赖这些异常
grep -rn "except BaseException" backend/
# 预期结果：空或仅与系统信号相关
```
