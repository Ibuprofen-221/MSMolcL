# 2.6 GPU Worker 进程静默崩溃检测滞后 — 解决方案

## 问题定位

`backend/services/retrieve_runtime.py` 中 Worker 存活检测仅在 `submit()` 和 `get_job()` 时触发：

```python
# backend/services/retrieve_runtime.py:306-314

def _sync_worker_state(self) -> None:
    for ion_mode, proc in self._worker_processes.items():
        if proc.is_alive():
            continue

        if self._worker_errors.get(ion_mode) is None and not self._stop_event.is_set():
            error = f"GPU检索进程异常退出({ion_mode})"
            self._worker_errors[ion_mode] = error
            self._mark_active_jobs_failed(error, ion_mode=ion_mode)
```

调用链：
```
submit() → _cleanup_finished_locked() → _sync_worker_state()
get_job() → _cleanup_finished_locked() → _sync_worker_state()
```

**如果系统空闲（无新任务提交、无状态轮询），Worker 崩溃将完全不被感知**，已提交的作业永久卡在 pending/running。

## 影响

- GPU Worker 因 OOM、CUDA 错误等原因崩溃后，用户作业永远无法完成
- 无告警、无自动恢复

## 解决方案

### 核心思路

添加独立的后台健康检查线程，定期（每 5 秒）检测 Worker 进程存活状态，并在检测到崩溃时：
1. 标记所有活跃作业为 failed
2. 记录错误日志
3. 自动尝试重启 Worker（可选，取决于是否有 GPU 资源泄漏风险）

### Step 1: 在 `RetrieveRuntime` 中添加健康检查线程

```python
# backend/services/retrieve_runtime.py（修改 RetrieveRuntime）

import logging
from threading import Event, Thread

logger = logging.getLogger(__name__)

HEALTH_CHECK_INTERVAL = 5        # 健康检查间隔（秒）
AUTO_RESTART_MAX_RETRIES = 2     # 自动重启最大尝试次数


class RetrieveRuntime:
    def __init__(self) -> None:
        # ... 原有字段 ...
        self._health_thread: Thread | None = None
        self._restart_attempts: dict[str, int] = {}  # ion_mode → 重启次数

    async def startup(self) -> None:
        if self._worker_processes and all(proc.is_alive() for proc in self._worker_processes.values()):
            return

        self._stop_event.clear()
        self._worker_errors = {mode: None for mode in VALID_ION_MODES}
        self._restart_attempts = {mode: 0 for mode in VALID_ION_MODES}

        # ... 原有 Worker 启动逻辑 ...

        # 新增：启动健康检查线程
        self._health_thread = Thread(
            target=self._health_loop,
            daemon=True,
            name="retrieve-health-check",
        )
        self._health_thread.start()

    async def shutdown(self) -> None:
        self._stop_event.set()
        # ... 原有 shutdown 逻辑 ...

    def _health_loop(self) -> None:
        """后台健康检查：定期检测 Worker 存活并尝试自动恢复。"""
        while not self._stop_event.is_set():
            self._stop_event.wait(timeout=HEALTH_CHECK_INTERVAL)
            if self._stop_event.is_set():
                return

            self._check_and_recover()

    def _check_and_recover(self) -> None:
        for ion_mode, proc in list(self._worker_processes.items()):
            if proc.is_alive():
                continue

            # Worker 已死
            error_msg = f"GPU检索进程异常退出({ion_mode})"
            logger.error(error_msg)

            self._worker_errors[ion_mode] = error_msg
            self._mark_active_jobs_failed(error_msg, ion_mode=ion_mode)

            # 尝试自动重启
            attempts = self._restart_attempts.get(ion_mode, 0)
            if attempts < AUTO_RESTART_MAX_RETRIES:
                logger.warning(
                    "尝试重启 Worker(%s)，第 %d/%d 次",
                    ion_mode, attempts + 1, AUTO_RESTART_MAX_RETRIES,
                )
                success = self._restart_worker(ion_mode)
                if success:
                    self._restart_attempts[ion_mode] = attempts + 1
                    self._worker_errors[ion_mode] = None

    def _restart_worker(self, ion_mode: str) -> bool:
        """重启单个 ion_mode 的 Worker 进程。"""
        try:
            mp_ctx = multiprocessing.get_context(retrieve_mp_start_method)

            request_queue = mp_ctx.Queue(maxsize=retrieve_max_pending_jobs)
            response_queue = mp_ctx.Queue(maxsize=retrieve_max_pending_jobs * 2)

            process = mp_ctx.Process(
                target=_gpu_worker_main,
                args=(request_queue, response_queue, ion_mode),
                daemon=False,
                name=f"retrieve-gpu-worker-{ion_mode}",
            )
            process.start()

            # 替换旧的队列和进程（旧队列已不可用）
            self._request_queues[ion_mode] = request_queue
            self._response_queues[ion_mode] = response_queue
            self._worker_processes[ion_mode] = process

            logger.info("Worker(%s) 重启成功", ion_mode)
            return True
        except Exception as exc:
            logger.error("Worker(%s) 重启失败: %s", ion_mode, exc)
            return False
```

### Step 2: 同样修改 `RetrieveAdvancedRuntime`

`backend/services/retrieve_advanced_runtime.py` 中存在相同问题，需做相同改造。由于两个类结构高度相似，建议：

**短期**：复制同样的 `_health_loop` / `_check_and_recover` / `_restart_worker` 到 `RetrieveAdvancedRuntime`。

**长期**（见 1.5 重复代码解决方案）：提取公共基类 `BaseRetrieveRuntime`，避免重复实现。
