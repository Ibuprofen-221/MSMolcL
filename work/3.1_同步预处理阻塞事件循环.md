# 3.1 同步预处理阻塞事件循环 — 解决方案

## 问题定位

`backend/api/file_upload.py` 在 async 请求处理函数中同步调用 `preprocess_main()`：

```python
# backend/api/file_upload.py:238
preprocess_main(str(saved_mgf_path), str(saved_json_path), output_base_dir=pair_dir)
```

`preprocess_main()` → `SpectrumNormalizer.parse_spectrum_file()` → 全量读取 MGF 文件 + NumPy 数组操作 + JSON 序列化。

在批量上传场景下，for 循环中串行调用：

```python
# backend/api/file_upload.py:210-238
for idx, stem in enumerate(paired_stems, start=1):
    # ...
    await save_upload_file(...)       # async I/O
    await save_upload_file(...)       # async I/O
    preprocess_main(...)              # ← 同步阻塞！整个 event loop 在此挂起
    # ...
```

同样 `backend/services/sirius_batch_service.py:214` 在聚合阶段也有同步 `preprocess_main()` 调用。

## 影响

- 预处理期间（可能数秒到数十秒），所有其他 API 请求被阻塞
- 包括 health check，可能导致负载均衡器误判服务不可用
- 并发上传时，后一个请求需等待前一个完成预处理

## 解决方案

### Step 1: 将预处理移到线程池执行

FastAPI / Starlette 提供了 `run_in_executor`，将同步调用调度到默认线程池：

```python
# backend/api/file_upload.py（修改后）

import asyncio

# ...在循环中...
for idx, stem in enumerate(paired_stems, start=1):
    # ...文件保存...

    # 修改前
    # preprocess_main(str(saved_mgf_path), str(saved_json_path), output_base_dir=pair_dir)

    # 修改后：在线程池中执行
    loop = asyncio.get_running_loop()
    await loop.run_in_executor(
        None,  # 使用默认 ThreadPoolExecutor
        preprocess_main,
        str(saved_mgf_path),
        str(saved_json_path),
        pair_dir,  # output_base_dir
    )

    # 后续的文件读取和合并逻辑可以保持不变，因为它们依赖预处理结果
```

### Step 2: 并行处理多个文件对

将串行 for 循环改为并发执行，进一步缩短总耗时：

```python
# backend/api/file_upload.py（进一步优化）

import asyncio
from concurrent.futures import ThreadPoolExecutor

async def _process_one_pair(
    stem: str,
    mgf_file: UploadFile,
    json_file: UploadFile,
    pair_dir: Path,
    current_user: User,
    task_id: str,
) -> dict:
    """处理单个文件对（在线程池中执行预处理）。"""
    # ...文件保存逻辑...

    loop = asyncio.get_running_loop()
    await loop.run_in_executor(
        None,
        preprocess_main,
        str(saved_mgf_path),
        str(saved_json_path),
        pair_dir,
    )

    # ...读取并返回 pair 结果...
    return {
        "pair_key": stem,
        "pair_dir": pair_dir.name,
        "root_info": pair_root_info,
        "valid_pairs": pair_valid_pairs,
        "fragtrees": pair_fragtrees,
        "spectra_text": pair_spectra_text,
    }


# 在 handle 函数中使用 asyncio.gather 并行处理
async def _process_batch(
    paired_stems: list[str],
    mgf_stem_map: dict,
    json_stem_map: dict,
    task_dir: Path,
    current_user: User,
    task_id: str,
) -> list[dict]:
    tasks = []
    for idx, stem in enumerate(paired_stems, start=1):
        pair_dir = task_dir / f"pair_{idx:03d}_{stem}"
        pair_dir.mkdir(parents=True, exist_ok=True)

        tasks.append(
            _process_one_pair(
                stem,
                mgf_stem_map[stem],
                json_stem_map[stem],
                pair_dir,
                current_user,
                task_id,
            )
        )

    # 并发执行所有文件对的预处理（限制最大并发数）
    semaphore = asyncio.Semaphore(4)  # 最多 4 个并行预处理任务

    async def bounded(task):
        async with semaphore:
            return await task

    return await asyncio.gather(*[bounded(t) for t in tasks])
```

由于 `sirius_batch_service.py` 中的聚合调用链较复杂（被同步函数 `get_mgf_only_batch_status` 调用，而后者又被 async handler 调用），需做如下改造：

```python
# backend/services/sirius_batch_service.py（新增）

import asyncio

# 聚合任务追踪
_aggregate_tasks: dict[str, asyncio.Task] = {}


async def trigger_aggregate_async(user_data_path: str, task_id: str) -> None:
    """异步触发聚合（不阻塞调用方）。"""
    task_key = f"{user_data_path}:{task_id}"
    if task_key in _aggregate_tasks:
        return  # 已在聚合中

    async def _do_aggregate():
        try:
            task_dir = get_user_task_dir(user_data_path=user_data_path, task_id=task_id, create=False)
            meta = _read_meta(task_dir)
            loop = asyncio.get_running_loop()

            # 在线程池中执行聚合（因为 preprocess_main 是同步 CPU 操作）
            await loop.run_in_executor(
                None,
                _run_aggregate_if_needed_sync,
                user_data_path, task_id, task_dir, meta,
            )
        finally:
            _aggregate_tasks.pop(task_key, None)

    _aggregate_tasks[task_key] = asyncio.create_task(_do_aggregate())


def _run_aggregate_if_needed_sync(user_data_path, task_id, task_dir, meta):

    pass
```

然后在 API handler 中：

```python
# backend/api/file_upload_status.py（修改）

@file_upload_status_router.get("/upload-files/status")
async def upload_files_status_api(task_id, current_user):
    # 先获取状态
    data = get_mgf_only_batch_status(current_user.data_path, task_id)

    # 如果需要聚合，异步触发（不阻塞状态响应）
    if data.get("status") == "running" and not data.get("files"):
        await trigger_aggregate_async(current_user.data_path, task_id)

    return success_response(message="上传任务状态", data=data)
```
