Project Task:
请为我的项目编写一套完整的后端核心代码，实现支持多用户、数据隔离、以及基于 JWT 认证的用户管理系统。
# Core Architecture:
用户信息存储: 使用 SQLite 存储账号、哈希密码及用户元数据。
业务数据存储: 每个用户的具体业务数据以 JSON 文件 形式存放在服务器本地。
身份验证: 基于 JWT (JSON Web Tokens) 的 Bearer Token 机制。
# Technical Stack:
Framework: FastAPI
Database: SQLite + SQLAlchemy (ORM)
Security: PyJWT, Passlib (with bcrypt)
Validation: Pydantic Models
File System: Python os, pathlib 和 json
# Detailed Requirements:
## 模型设计 (Models):
User Model (DB): 包含 id, username, hashed_password, 和 data_path。
data_path 逻辑: 该字段存储用户专属数据目录的相对路径。
## Pydantic Schemas: 定义用户注册、登录、Token 返回以及业务数据交互的数据结构。
混合存储与隔离逻辑:
根目录: 设定 ./user_data/ 为所有用户数据的根目录。
注册初始化: 当 POST /register 被调用时：
后端使用 bcrypt 哈希加密密码。
为用户生成一个唯一的文件夹（使用用户名）。
在当前文件夹中存放和当前temp文件夹下一致的数据结构；
将此相对路径存入数据库的 data_path 字段。
安全防护 (Security):
密码安全: 实现 get_password_hash 和 verify_password 函数，绝不存储明文。
JWT 验证: 编写 get_current_user 依赖项。它必须从 Header 解析 Token，验证用户有效性，并返回 User 对象。
路径安全: 编写一个路径检查工具，确保合并后的绝对路径始终位于 ./user_data/ 内部，严防路径穿越攻击 (Path Traversal)。
# API Endpoints:
POST /register: 用户注册、初始化本地存储。
POST /login: 验证身份并发放 JWT。
GET /my-data: 核心接口。通过 get_current_user 获取当前用户，根据其数据库中的 data_path 读取物理 JSON 文件并返回内容。
PUT /my-data: 核心接口。接收 JSON 数据，验证格式后覆盖写入该用户的物理 JSON 文件。
# 代码风格:
包含完善的异常处理（如：用户已存在返回 400，认证失败返回 401，文件缺失返回 404 等）。
